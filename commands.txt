Документ содержит список всех выполненных команд и пояснения к ним.

1. Проверка текущей ветки:
git status

2. Просмотр истории коммитов:
git log --oneline

3. Добавление в staging area:
git add 

4. Коммит:
git commit -m "////"

5. Отправка ветки main на GitHub:
git push origin main

6. Создание ветки ...:
git branch ...

7. Проверка списка веток:
git branch

8. Переключение на ветку ...:
git switch ...

9. Просмотр историй всех веток:
git log --oneline --graph --all

10. Сравнение веток:
git diff mybranch main

11. Отправка ветки mybranch на GitHub:
git push -u origin mybranch

12. git restore -  позволяет откатить изменения в файлах и вернуть их к состоянию последнего коммита

13. Инициализация репозитория
git init

14.Слияние двух веток
git merge
